## 2025-03-02 基本概念

**来源**: [Prompt Engineering Guide - 基本概念](https://www.promptingguide.ai/zh/introduction/basics)

### 核心观点

1. **提示质量决定输出质量**：提供的信息越完善，结果越好
2. **提示词可包含多种元素**：指令、问题、上下文、输入、示例
3. **提示工程是设计有效提示词的方法**：明确指示模型完成任务
4. **零样本提示**：直接提问，不提供示例
5. **少样本提示**：提供示例，增强上下文学习能力

### 关键概念

| 术语 | 解释 |
|------|------|
| **Prompt（提示词）** | 传递给模型的指令、问题或其他信息 |
| **零样本提示** | 不提供示例，直接让模型回答问题 |
| **少样本提示** | 提供示例示范，帮助模型理解任务 |
| **上下文学习** | 模型基于示例学习和理解任务的能力 |
| **System/User/Assistant** | 聊天模型的三种角色，用于构建结构化对话 |

### 标准提示词格式

```
<指令>
<上下文>
<输入数据>
<输出指示>
```

### 问答格式（QA Format）

**零样本提示**：
```
Q: <问题>
A: <模型回答>
```

**少样本提示**：
```
Q: <示例问题1>
A: <示例答案1>

Q: <示例问题2>
A: <示例答案2>

Q: <实际问题>
A:
```

### 行动项

- [ ] 优化现有提示词：检查是否包含足够的上下文和明确指令
- [ ] 尝试少样本提示：当输出不理想时，提供 2-3 个示例
- [ ] 使用角色设定：利用 `system` 角色设定模型行为

## 2025-03-02 提示词要素

**来源**: [Prompt Engineering Guide - 提示词要素](https://www.promptingguide.ai/zh/introduction/elements)

### 核心观点

1. **提示词由四要素构成**：指令、上下文、输入数据、输出指示
2. **要素非必需**：根据任务类型灵活组合
3. **示例是最好的上下文**：帮助模型理解预期输出

### 关键概念

| 要素 | 说明 | 示例 |
|------|------|------|
| **指令** | 要模型执行的任务 | "将文本分类为..." |
| **上下文** | 外部信息或背景 | 额外的示例或说明 |
| **输入数据** | 用户输入的内容 | "我觉得食物还可以" |
| **输出指示** | 输出的类型或格式 | "情绪：" |

### 示例模板

```
请将文本分为中性、否定或肯定    ← 指令
文本：我觉得食物还可以。        ← 输入数据
情绪：                          ← 输出指示
```

## 2025-03-02 设计提示的通用技巧

**来源**: [Prompt Engineering Guide - 设计提示的通用技巧](https://www.promptingguide.ai/zh/introduction/tips)

### 核心观点

1. **迭代是关键**：从简单开始，逐步优化
2. **具体胜于模糊**：直接明确地说要做什么
3. **正向指令更有效**：说"要做什么"而非"不要做什么"

### 关键技巧

| 技巧 | 说明 | 示例 |
|------|------|------|
| **从简单开始** | 逐步添加元素，迭代优化 | 先写基础提示，再补充上下文 |
| **使用明确指令** | 写入、分类、总结、翻译等 | "总结以下文章" |
| **保持具体性** | 描述详细，但避免冗余 | 指定输出格式、长度、风格 |
| **避免不明确** | 直接了当，不要绕弯 | ❌ "解释一下提示工程" → ✅ "用2句话解释提示工程" |
| **正向表述** | 告诉模型做什么 | ❌ "不要推荐恐怖片" → ✅ "推荐喜剧或动作片" |

### 指令放置建议

```
### 指令 ###
将文本分类为积极、消极或中性

### 上下文 ###
这是一个产品评论分类任务

### 输入 ###
{text}
```

## 2025-03-02 提示词应用示例

**来源**: [Prompt Engineering Guide - 提示词示例](https://www.promptingguide.ai/zh/introduction/examples)

### 核心观点

1. **提示词可完成多种NLP任务**：概括、提取、问答、分类、对话、代码、推理
2. **角色提示控制风格**：设定模型身份和回答风格
3. **分步指令提升推理**：复杂任务拆解为步骤

### 应用场景速查

| 场景 | 提示模板 | 要点 |
|------|----------|------|
| **文本概括** | `用一句话解释上面的信息` | 指定长度限制 |
| **信息提取** | `提到的人名是：` | 明确提取目标 |
| **问答** | `基于上下文回答，不确定时回复"不确定"` | 添加约束条件 |
| **文本分类** | 提供示例 + `情感：` | 用示例指定输出格式 |
| **对话/角色** | `助手回答应该易于小学生理解` | 设定目标受众 |
| **代码生成** | `/* 注释描述功能 */` | 用注释描述需求 |
| **推理** | `分步解决：先识别奇数，再求和` | 拆解复杂任务 |

### 角色提示示例

```
以下是与AI研究助手的对话。助手回答应该易于小学生理解。

Human: 你能解释黑洞的形成吗？
AI:
```

### 推理任务优化

```
❌ 这组奇数之和是偶数：15, 32, 5, 13, 82, 7, 1

✅ 分步解决：
1. 首先识别奇数
2. 计算它们的和
3. 判断结果是奇数还是偶数
```

## 2025-03-02 提示技术：零样本 & 少样本

**来源**:
- [零样本提示](https://www.promptingguide.ai/zh/techniques/zeroshot)
- [少样本提示](https://www.promptingguide.ai/zh/techniques/fewshot)

### 核心观点

1. **零样本依赖模型能力**：无需示例，直接指令
2. **少样本启用上下文学习**：通过演示引导模型
3. **格式比标签更重要**：即使随机标签也有效
4. **复杂推理需要更高级技术**：少样本对推理任务效果有限

### 技术对比

| 特性 | 零样本 | 少样本 |
|------|--------|--------|
| **示例** | 无 | 1-多个 |
| **依赖** | 模型预训练知识 | 上下文学习能力 |
| **适用场景** | 简单分类、翻译 | 新词用法、特定格式 |
| **局限性** | 复杂任务效果差 | 推理任务仍不足 |

### 少样本提示要点

```
✅ 格式一致很重要
✅ 标签分布应接近真实
✅ 随机标签也比无标签好
❌ 对复杂推理任务效果有限
```

### 示例

**零样本**：
```
将文本分类为中性、负面或正面。
文本：我认为这次假期还可以。
情感：
```

**少样本**：
```
这太棒了！// Negative（随机标签也有效）
这太糟糕了！// Positive
哇，那部电影太棒了！// Positive
多么可怕的节目！//
```

## 2025-03-02 提示技术：链式思考（CoT）

**来源**: [链式思考提示](https://www.promptingguide.ai/zh/techniques/cot)

### 核心观点

1. **中间步骤提升推理**：展示推理过程而非仅给答案
2. **零样本CoT极其简单**：添加"让我们逐步思考"
3. **Auto-CoT自动化**：让模型自动生成推理链示例
4. **大模型涌现能力**：需要足够大的模型才有效

### 三种CoT方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **少样本CoT** | 提供带推理步骤的示例 | 有高质量示例时 |
| **零样本CoT** | 添加"让我们逐步思考" | 无示例可用时 |
| **Auto-CoT** | 模型自动生成推理链 | 需要大量示例时 |

### 零样本CoT示例

```
❌ 我去市场买了10个苹果。我给了邻居2个苹果和修理工2个苹果。
   然后我去买了5个苹果并吃了1个。我还剩下多少苹果？

✅ 我去市场买了10个苹果。我给了邻居2个苹果和修理工2个苹果。
   然后我去买了5个苹果并吃了1个。我还剩下多少苹果？

   让我们逐步思考。
```

**输出**：
```
首先，您从10个苹果开始。
您给了邻居和修理工各2个苹果，所以您还剩下6个苹果。
然后您买了5个苹果，所以现在您有11个苹果。
最后，您吃了1个苹果，所以您还剩下10个苹果。
```

### 少样本CoT示例

```
这组数中的奇数加起来是偶数：4、8、9、15、12、2、1。
A：将所有奇数相加（9、15、1）得到25。答案为False。

这组数中的奇数加起来是偶数：15、32、5、13、82、7、1。
A：
```

## 2025-03-02 提示技术：自我一致性

**来源**: [自我一致性](https://www.promptingguide.ai/zh/techniques/consistency)

### 核心观点

1. **多路径采样**：生成多个推理路径和答案
2. **多数投票**：选择最一致的答案
3. **与CoT结合**：增强算术和常识推理
4. **代价是计算量**：需要多次调用模型

### 工作流程

```
问题 → 多次采样（CoT）→ 多个推理路径 → 多个答案 → 多数投票 → 最终答案
```

### 示例

**问题**：
```
当我6岁时，我的妹妹是我的一半年龄。现在我70岁了，我的妹妹多大？
```

**多次采样结果**：
| 路径 | 推理 | 答案 |
|------|------|------|
| 1 | 妹妹当时3岁，现在70-3=67 | 67 |
| 2 | 妹妹当时3岁，现在70-3=67 | 67 |
| 3 | 妹妹当时3岁，现在70/2=35 | 35 |

**最终答案**：67（多数一致）

### 适用场景

- 算术推理
- 常识推理
- 需要高准确率的场景
- 可接受多次调用的成本
