- **Vibe Coding 开发心得：循序渐进**
  - 背景：Vibe Coding（氛围编程/直觉编程）是一种依赖 AI 辅助的编程方式
  - 核心体会：**需要循序渐进地进行功能开发**

  - **任务开始前要思考清楚**：
    1. 做事情的顺序是什么？（先后依赖关系）
    2. 每一步逻辑做完后，如何交付 AI 进行验证？
    3. 交付物如何传递给下一步？

  - 原因分析：
    - AI 一次生成大量代码容易出现幻觉、遗漏细节
    - 功能堆叠过快导致难以定位问题
    - 缺乏中间验证点，出问题后难以回溯

  - **最佳实践**：
    - 小步快跑：一次只做一个功能点
    - 即时验证：每完成一个点立即测试
    - 保存快照：关键节点 commit，方便回退
    - 明确边界：告诉 AI "只做这个，其他不动"
    - **定义交付物**：每步产出什么，下一步如何接收

  - **示例流程**：
    ```
    Step 1: 数据模型定义
      ↓ 交付物：Pydantic Model
    Step 2: AI 验证 Model 是否符合需求
      ↓ 交付物：验证通过的 Model
    Step 3: 基于 Model 实现 API
      ↓ 交付物：API 接口
    Step 4: AI 验证 API 是否正确使用 Model
      ↓ 交付物：验证通过的 API
    ...
    ```

  - 与 dev-flow 的关系：
    - dev-flow 是完整流程（复杂功能）
    - vibe coding 的循序渐进是日常习惯（简单到中等功能）
    - 两者互补：复杂用 dev-flow，简单用循序渐进
