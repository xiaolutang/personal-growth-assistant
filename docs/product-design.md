## 一、产品定位

### 1.1 核心口号

**过程属于自己** - 在 AI 时代，工作成果属于公司，但成长过程、学习笔记、决策思考才是真正属于你的财富。

### 1.2 目标用户

| 用户类型 | 入口 | 核心价值 | 付费意愿 |
|---------|------|---------|---------|
| **技术用户** | Claude Code / Cursor（MCP） | AI 直接操作数据，无缝融入工作流 | 高 |
| **知识工作者** | Web App | 知识碎片化、项目多，需要统一管理 | 高 |
| **终身学习者** | Web App | 学了记不住，需要智能检索 | 中 |
| **自由职业者** | Web App | 时间管理、项目追踪 | 高 |

### 1.3 产品形态

**双入口并行**：MCP Server（技术用户） + Web App（普通用户）

### 1.4 核心场景（按优先级）

| 优先级 | 场景 | 描述 | 技术支撑 |
|--------|------|------|---------|
| P0 | **快速记录** | 说一句话，AI 自动分类存入 | LLM 解析 + Markdown 写入 |
| P0 | **智能检索** | "上周关于 RAG 的笔记" | RAG（Qdrant） |
| P1 | **成长回顾** | "我这周干了啥" | LLM 总结 + 时间维度 |
| P2 | **知识关联** | "这个概念和什么相关" | 知识图谱（Neo4j） |

### 1.5 差异化卖点

| 卖点 | 说明 | 竞品对比 |
|------|------|---------|
| 本地 Markdown 存储 | 数据透明，用户可直接编辑 | Mem/Reflect 是黑盒 |
| 原生 MCP 支持 | Claude Code/Cursor 可直接操作 | 独一份 |
| AI First | 自动整理、智能检索 | 国内产品 AI 弱 |
| 个人成长场景 | 任务+笔记+项目+决策闭环 | 通用笔记无此场景 |

## 二、技术架构

### 2.1 存储层（四层）

```
Markdown（Source of Truth）
    ↓
SQLite（索引 + 全文搜索）
    ↓
Qdrant（向量检索）
    ↓
Neo4j（知识图谱）
```

### 2.2 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | FastAPI + Python 3.11 |
| 存储 | Markdown + SQLite + Qdrant + Neo4j |
| MCP | mcp SDK（Python） |
| 前端 | React + TypeScript + shadcn/ui |
| LLM | OpenAI 兼容 API |

## 五、关键设计决策

### 5.1 数据关联设计

| 设计点 | 决策 |
|--------|------|
| **笔记引用** | Wiki 链接语法（`[[note-id]]`） |
| **父子关系** | 单父级（parent_id），后续可扩展为多父级 |
| **引用解析** | 解析 Markdown 中的 `[[note-id]]` 建立引用关系 |

### 5.2 AI 能力边界

| 能力 | 权限 |
|------|------|
| **解析/创建/检索** | ✅ 允许 |
| **修改数据** | ✅ 允许（需告知） |
| **删除数据** | ❌ 禁止（必须用户确认） |
| **不确定时** | 询问用户 |

### 5.3 用户流程设计

| 场景 | 设计 |
|------|------|
| **首次使用** | 提供模板（项目模板、任务模板） |
| **日常入口** | 今日任务优先 |
| **快速记录** | AI 解析 → 确认/修改 → 保存 |

### 5.4 数据同步机制

| 设计点 | 决策 |
|--------|------|
| **外部修改检测** | 实时监听文件变化 |
| **同步顺序** | Markdown → SQLite → Neo4j/Qdrant（并行） |
| **同步失败** | 异步重试（记录失败，后续重试） |

## 七、用户场景

### 场景1：快速记录想法

> 用户：今天学了 RAG，重点是向量检索和 Embedding
>
> 系统：自动创建笔记，提取标签 #RAG #向量检索 #Embedding，关联到学习项目

### 场景2：查找历史内容

> 用户：上周关于 RAG 的笔记是什么？
>
> 系统：通过语义搜索找到相关笔记，展示内容

### 场景3：时间统计分析

> 用户：我最近时间花哪了？
>
> 系统：统计最近 30 天的任务完成情况，按项目/类别分组展示

### 场景4：AI 工具集成

> 用户（在 Claude Code 中）：帮我更新学习项目的进度
>
> Claude Code：通过 MCP 调用个人成长助手，修改对应 Markdown 文件
