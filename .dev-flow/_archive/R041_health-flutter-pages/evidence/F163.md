# F163: 修复 eslint-disable react-hooks/exhaustive-deps

## 审查结果

共审查 5 处 eslint-disable，修复 2 处（移除抑制），保留 3 处（添加原因注释）。

### 1. App.tsx (已修复 - 移除抑制)

**位置**: `useEffect(() => { loadFromStorage(); }, [])`
**问题**: `loadFromStorage` 是 zustand store 的 action，引用稳定，加入依赖不会导致额外重渲染。
**修复**: 将 `loadFromStorage` 加入依赖数组，移除 eslint-disable。

### 2. TimelineView.tsx (保留 - 添加注释)

**位置**: `useCallback` 的依赖数组 `[goalsIdKey]`
**原因**: `goalsIdKey` 是 goals 列表 ID 的序列化字符串，故意用它替代 `goals` 数组引用，避免 goals 引用变化（但内容不变）时触发不必要的重新 fetch。
**修复**: 将 eslint-disable 移至依赖行上方，添加 `-- goalsIdKey 是 goals id 的稳定序列化` 注释。

### 3. useTaskFilters.ts (保留 - 添加注释)

**位置**: `useEffect(() => { useTaskStore.getState()... }, [])`
**原因**: 使用 `useTaskStore.getState()` 获取 store 快照，仅在组件挂载时执行一次性检查，不响应 reactive 变化。
**修复**: 添加 `-- 挂载时一次性检查 store 快照，无需响应 reactive 变化` 注释。

### 4. useExploreSearch.ts (保留 - 添加注释)

**位置**: `useEffect` 中 `if (t !== activeTab) setActiveTab(t)`
**原因**: 仅在 URL 参数（searchParams）变化时同步覆盖本地 activeTab 状态，故意不依赖 activeTab 以避免循环更新。
**修复**: 添加 `-- 仅在 URL 参数变化时同步覆盖本地 state，故意不依赖 activeTab 以避免循环` 注释。

### 5. PageChatPanel.tsx (已修复 - 移除抑制)

**位置**: `useEffect` 加载历史消息
**问题**: effect 内部使用了 `greetingMessage` 和 `maxMessages`，但依赖数组只有 `pageContext?.page`。
**修复**: 将 `greetingMessage` 和 `maxMessages` 加入依赖数组，移除 eslint-disable。`messages.length` 未加入依赖是合理的（page 变化时总是重新初始化）。
