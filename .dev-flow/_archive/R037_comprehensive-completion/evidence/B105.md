## 实现摘要

扩展 GET /entries 路由，新增 `due` 查询参数支持到期过滤：
- `due=today` — 返回 planned_date 为今天（UTC 日期）的任务
- `due=overdue` — 返回 planned_date 早于今天且 status != complete 的任务
- 日界规则统一 UTC midnight，使用 `DATE()` SQLite 函数处理 ISO 格式存储

不新增数据库列，复用已有 `planned_date` 字段。不修改 notification_service.py 和 useHomeData.ts。

## 实现细节

### SQLite 查询逻辑
```sql
-- due=today
WHERE DATE(e.planned_date) = '<today_utc>'

-- due=overdue
WHERE e.planned_date IS NOT NULL
  AND e.planned_date != ''
  AND DATE(e.planned_date) < '<today_utc>'
  AND e.status != 'complete'
```

`planned_date` 在 SQLite 中以 ISO 格式存储（如 `2026-04-26T00:00:00`），使用 `DATE()` 函数提取日期部分进行比较，确保跨时间存储格式的兼容性。
