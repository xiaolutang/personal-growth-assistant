### 实现变更

**修改文件：**
- `frontend/src/pages/explore/useExploreSearch.ts` — 核心变更
  - `filteredTasks` 逻辑：搜索模式下使用 `filterByCategory(searchResults, "")` 仅过滤 Explore 类别边界（排除 task 等），不做 activeTab 二次过滤，实现跨类型混合展示
  - 非搜索模式保持原有 `filterByCategory(entries, activeTab)` 按 Tab 过滤
  - 两处搜索调用（防抖 + 手动）均传递 `activeTab || undefined` 作为 filter_type

**新增测试：**
- `frontend/src/pages/explore/__tests__/useExploreSearch.test.ts` — 6 个 hook 测试

  - 全部 Tab 不传 filter_type（undefined）
  - 搜索结果过滤 Explore 类别边界（排除 task 类型）
  - 非搜索模式 Tab 过滤正常工作
  - 笔记 Tab 传递 filter_type=note
- `frontend/src/pages/explore/__tests__/utils.test.ts` — 3 个新增 utils 测试
  - tab 为空时排除 task 等非 Explore 类型
  - tab 为空时保留所有 Explore 类型（混合展示）
  - 空数组返回空

### systemic_preflight

- **reuse_candidates**: filterByCategory 已有函数，搜索模式复用空 tab 参数实现边界过滤，无新增抽象
- **adjacent_impact**: 非搜索模式 Tab 过滤逻辑未改动，Explore 页其余功能（批量操作、列表渲染）不受影响
- **pattern_consistency**: activeTab || undefined 转换与现有 searchEntries API 签名一致，无需类型扩展

### behavior_description

1. **Tab 切换透传**：用户切换到"灵感/笔记/项目"等 Tab 时，`activeTab` 值通过 `searchEntries(query, limit, activeTab || undefined, filters)` 传递给后端 `/search` API 的 `filter_type` 参数。"全部" Tab 的 `activeTab` 为空字符串，经 `|| undefined` 转换后不传 `filter_type`。
2. **搜索结果跨类型混合展示**：搜索模式下（`searchResults !== null`），`filteredTasks` 使用 `filterByCategory(searchResults, "")` 仅过滤 Explore 类别边界（排除 task 等），不按 activeTab 二次过滤。后端已按 filter_type 过滤了类型，前端不再重复过滤。
3. **非搜索模式 Tab 过滤**：非搜索模式下（`searchResults === null`），`filteredTasks` 使用 `filterByCategory(entries, activeTab)` 正常按 Tab 过滤。
