# F06 - 页面转场动画

## 变更文件

- `mobile/lib/config/routes.dart`

## 实现内容

### 1. EntryDetail (`/entries/:id`) — 平台感知转场
- `builder:` 改为 `pageBuilder:`
- **iOS**: 使用 `CupertinoPage`，保留原生 iOS slide 动画 + swipe back 手势
- **Android**: 使用 `CustomTransitionPage` + SlideTransition，从右滑入，Curves.easeInOut

### 2. GoalDetail (`/goals/:id`) — 平台感知转场
- 同 EntryDetail，iOS 用 CupertinoPage，Android 用 CustomTransitionPage slide

### 3. Settings (`/settings`) — Fade 转场（跨平台统一）
- `builder:` 改为 `pageBuilder:`，使用 `CustomTransitionPage`
- FadeTransition: opacity 跟随 animation
- 设置页统一 fade（不需要 swipe back 手势）

### 4. ShellRoute 内主页面 — 保持不变
- `/`, `/chat`, `/explore`, `/tasks`, `/notes`, `/inbox`, `/review`, `/goals`
- 全部保持原有 `builder:` 写法，使用系统默认转场

## 关键设计决策

- 详情页（EntryDetail、GoalDetail）使用平台分流：iOS 保留原生体验（含 swipe back），Android 使用自定义 slide
- 设置页统一使用 fade 过渡（不需要 swipe back）
- 三个路由均在 ShellRoute 内部，自定义转场不与 Shell 动画冲突
- 使用 `Platform.isIOS` 判断平台，引入 `dart:io` 和 `cupertino.dart`
