## Architecture Constraints

- Call direction: routers -> services -> infrastructure, strictly one-way
- category_group maps to a set of Category enum values at the service layer
- Must be compatible with existing type, status, priority, tags filters
- CATEGORY_GROUPS: {"actionable": {"task","decision","project"}, "knowledge": {"inbox","note","reflection","question"}}

## Acceptance Criteria

1. GET /entries?category_group=actionable 返回 category 为 task/decision/project 的条目
2. GET /entries?category_group=knowledge 返回 category 为 inbox/note/reflection/question 的条目
3. category_group 与 type 参数互斥，同时传时返回 422
4. 不传 category_group 和 type 时行为不变（返回全部）
5. category_group 与 status/priority/tags/sort_by 等参数可组合使用
6. 执行 npm run gen:types 成功，前端 client 包含 category_group 参数类型

## Contract Refs
