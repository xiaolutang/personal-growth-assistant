# F04 执行证据

| Task ID | F04 |
| Phase | P4 |
| Name | Decision 专属卡片 UI + 决策动作流 |

## 实现摘要

- 新建 `frontend/src/pages/tasks/DecisionCard.tsx`
  - decision 类型条目的专属卡片组件，显示标题 + content 摘要（line-clamp-2 截断）
  - 未决定状态显示三个按钮：决定 YES（绿色）/ 决定 NO（红色）/ 延后（灰色）
  - 已完成状态显示决策结果（YES/NO + 理由），隐藏操作按钮
  - 已延后状态显示「已延后」提示
  - 点击 YES/NO 弹出 DecisionResultDialog，输入理由后更新 status 为 complete，content 追加 `## 决策结果` 段落
  - 延后直接更新 status 为 paused，不弹对话框
  - 支持勾选「同时创建任务」，输入任务标题后创建 task 类型条目，parent_id 优先使用 decision.parent_id，否则使用 decision.id
- 新建 `frontend/src/pages/tasks/DecisionResultDialog.tsx`
  - 模态对话框：决策理由文本框（可选）+「同时创建任务」checkbox + 任务标题输入
  - 确认后调用回调函数处理决策逻辑
- 修改 `frontend/src/components/TaskList.tsx`
  - 条件渲染：category === 'decision' 时使用 DecisionCard，其他使用 TaskCard
- 新建 `frontend/src/pages/tasks/__tests__/DecisionCard.test.tsx`
  - 14 个测试用例覆盖所有验收标准
