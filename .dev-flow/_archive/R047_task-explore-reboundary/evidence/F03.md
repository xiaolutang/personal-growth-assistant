## 实现摘要

### 1. constants.ts — 新增子 Tab 定义
- `TASK_QUERY_PARAMS` 从 `{ type: "task", limit: 100 }` 改为 `{ category_group: "actionable", limit: 100 }`
- 新增 `TASK_SUB_TABS` 数组: 全部/任务/决策/项目
- 新增 `SubTabKey` 类型
- 新增 `ACTIONABLE_CATEGORIES` 常量

### 2. api.ts — getEntries 支持 category_group
- 参数类型新增 `category_group?: string`
- query 参数传递 `category_group`

### 3. taskStore.ts — fetchEntries 参数类型扩展
- 接口新增 `category_group?: string` 参数

### 4. useTaskFilters.ts — 重构核心逻辑
- 新增 `activeSubTab` / `setActiveSubTab` 状态
- 子 Tab 同步到 URL 参数 `?tab=task|decision|project`
- URL 参数恢复: `?tab=decision` 刷新后恢复正确子 Tab
- filteredTasks 增加 actionable category 过滤 + 子 Tab category 过滤
- 移除 `handleBatchCategory` 函数（不再支持转笔记/转灵感）
- `clearFilters` 同时重置 activeSubTab 为 "all"

### 5. Tasks.tsx — 页面 UI 变更
- 新增类型子 Tab 栏（全部/任务/决策/项目）
- 移除批量操作中的「转笔记」「转灵感」按钮，只保留「删除」
- 返回 100 条时底部显示「可能还有更多条目」提示
- 路由切换回 /tasks 时自动刷新数据
