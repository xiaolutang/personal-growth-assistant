## Architecture Constraints

- JWT secret 必须通过环境变量配置，不硬编码
- 密码使用 bcrypt 哈希
- 不做 refresh_token，仅 access_token（7天过期）
- login 失败返回 401（不暴露是用户名还是密码错误）
- 不做 get_optional_current_user

## Acceptance Criteria

1. POST /auth/register — 接收 username/email/password，创建用户，返回 201 + user info
2. POST /auth/login — 验证凭证，返回 access_token（7天过期）
3. POST /auth/logout — 清除 token（前端同步清除 localStorage）
4. GET /auth/me — 返回当前用户信息（需 Authorization header）
5. register 的 email/password 基础校验
6. login 失败返回 401
