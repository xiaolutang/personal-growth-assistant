## Implementation Summary

### 1. 日期解析纯函数 `parseDateKeyword(text)`
- 位于 `frontend/src/lib/dateParser.ts`
- 返回 `ParseResult | null`（含 `date` 和 `keyword`），保证 hint 文本与解析结果一致
- 纯函数，无副作用，不依赖 React
- 支持关键词：今天、明天、后天、大后天、本周X、下周X、下个月

### 2. 共享 Hook `useSmartSuggestions({ enableTypeSuggestion })`
- 位于 `frontend/src/lib/useSmartSuggestions.ts`
- 统一管理日期解析、类型建议、suppress 状态、stale-date 清理
- `autoFillEvent` state 通知消费者自动填充/清除（消费者在 useEffect 中监听）
- suppress 机制：手动清除或手动修改日期后不再自动填充，直到 reset
- reset() 清除所有状态（含 autoFillEvent），防止旧事件泄漏

### 3. CreateDialog 集成
- useEffect 监听 autoFillEvent 和 selectedType 变化
- 支持"先输入标题后选类型"场景（selectedType 变为 task 时应用已有 suggestedDate）
- handleTypeChange 清除 typeSuggestion（避免类型切换后残留旧建议）
- 日期高亮提示：`dateHint && plannedDate` 时显示

### 4. QuickCaptureBar 集成
- useEffect 监听 autoFillEvent（仅在 expanded 模式下生效）
- 展开时仅处理"先输入后展开"场景（通过 prevExpandedRef 避免重复解析）
- 提交成功后 reset（清除 suppress 和 autoFillEvent）
