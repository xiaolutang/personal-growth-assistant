## Design Decisions

1. **Two-mode design**: 默认模式创建 inbox（最快路径），展开模式创建 task（带优先级+日期），其他类型通过 CreateDialog。避免在 QuickCaptureBar 引入类型选择器，保持 UI 简洁。

2. **IME safety**: handleKeyDown 中检查 `e.nativeEvent.isComposing`，防止中文/日文输入法在组合确认时误触发创建。这与 CreateDialog 的回车快速提交保持一致的安全模式。

3. **FocusTrigger pattern**: 使用递增数字作为 focusTrigger，而非直接 ref 暴露。符合 React 单向数据流，Home.tsx 只需 `setFocusTrigger(prev => prev + 1)` 即可触发聚焦。

4. **Visual distinction**: QuickCaptureBar 使用实线边框(rounded-xl border)，无 AI 装饰元素（无渐变、无闪光动画），与 AgentChat/FloatingChat 视觉明确区分。

5. **Expand collapse on submit**: 展开模式提交后自动收起，减少用户下次操作的点击步骤。
