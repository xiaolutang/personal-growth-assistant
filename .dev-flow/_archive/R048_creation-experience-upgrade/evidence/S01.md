## Implementation Summary

构建了可复用的 `CreateDialog` 组件，支持全部 7 种条目类型的创建。

### Files Changed
- `frontend/src/components/CreateDialog.tsx` — 组件实现（新增）

### Key Design Decisions
1. **`<dialog>` 原生元素** — 与 ExportDialog、ConvertDialog 保持一致
2. **`categoryConfig` 单一权威来源** — ALL_CATEGORIES 从 `Object.keys(categoryConfig)` 派生，消除硬编码
3. **`defaultType` 真正可选** — 不传时 `selectedType` 为 `null`，不预选任何类型
4. **Enter 快速提交仅限 inbox/task** — `isQuickCreateCategory()` 守卫，非快速类型 Enter 不触发提交
5. **`allowedTypes` 约束** — `defaultType` 不在 `allowedTypes` 内时回退为 null
6. **移动端适配** — `max-sm:` 前缀实现底部抽屉式全屏展示

### Props API
```typescript
interface CreateDialogProps {
  open: boolean;

  onSuccess?: (entry: Task) => void;
```
