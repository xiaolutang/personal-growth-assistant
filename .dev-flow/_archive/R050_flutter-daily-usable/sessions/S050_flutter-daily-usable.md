## 需求来源

用户提出「把 Flutter 移动端核心页面补齐，做到日常使用的程度」。

## 头脑风暴结论

### 产品视角补充（已采纳）

1. **日循环设计**：首页晨报覆盖早晨，FAB 捕获覆盖白天，回顾页覆盖晚上
2. **快速捕获是核心**：全局 FAB → 底部弹窗 → 存 inbox（不调 AI 自动分类，直接存）
3. **注册非孤立**：需要忘记密码提示（简化为联系管理员）和空态引导（本轮不单独做，随页面自然呈现）
4. **设置页精简**：只做退出登录 + 通知开关

### 过度设计（已砍掉）

- 首页时段适配（早/晚不同内容）
- AI 自动分类捕获内容
- Tab 布局重设计
- 每日提醒/晨报推送/里程碑通知（只做到期通知）
- 完整空态引导
- 完整设置页

### 架构确认

- 后端 API 已全部就绪，无需后端改动
- forgot-password 简化处理（前端只显示提示文字）
- 新增依赖：flutter_local_notifications
