# B195 - 集成到 run_eval.py

## 任务描述
修改 run_eval.py，在评估完成后自动调用 report_generator 生成 HTML 报告并保存到 data/eval_reports/YYYY-MM-DD_HHMMSS.html，同时追加 history.json。添加 --report-dir 参数。

## 变更文件
- `backend/tests/eval/run_eval.py` - 主修改文件
- `backend/tests/eval/test_run_eval_report.py` - 新增测试文件

## 实现细节

### 1. run_single_turn() 返回值重构
- 原返回: `EvaluationReport`
- 新返回: `Tuple[EvaluationReport, List[Dict[str, Any]]]`
- 每个 per-case record: `{input, expected_tools, actual_tools, agent_reply, passed, category, elapsed_seconds}`
- agent_reply 来源: `_call_cache.get(thread_id, {}).get("content", "")`

### 2. run_negative() 返回值重构
- 原返回: `NegativeReport`
- 新返回: `Tuple[NegativeReport, List[Dict[str, Any]]]`
- 每个 per-case record: `{input, should_not_call, actual_tools, agent_reply, violated, violated_tools, category, elapsed_seconds}`
- agent_reply 来源: `result.get("content", "")`

### 3. 新增辅助函数
- `_get_env_info()`: 获取 git commit hash + LLM_MODEL，失败降级为 'unknown'
- `_get_default_report_dir()`: 解析项目根目录/data/eval_reports/

### 4. main() 增强
- 添加 `--report-dir` 参数
- 评估完成后调用 `build_report_data()` + `generate_html_report()` + `append_history()`
- HTML 报告保存到 `{report-dir}/YYYY-MM-DD_HHMMSS.html`
- history.json 追加到同目录
- 终端输出报告保存路径
- --output JSON 兼容性不受影响

### 5. 错误处理
- git rev-parse 失败: 降级为 'unknown'
- LLM_MODEL 未设置: 降级为 'unknown'
- 模板渲染失败: 终端 warning，不删除已生成的 history.json
- history.json 损坏: 备份为 .bak 并重建
