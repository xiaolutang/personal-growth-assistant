# F05: taskStore 性能优化 + React.memo

## 变更摘要

taskStore 存在 `isFetching` + `isLoading` 双 loading 标志冗余，且 TaskCard 订阅整个 tasks 数组仅用于查找 parentProject，导致任何 tasks 变更触发所有订阅者重渲染。

## 改动文件

| 文件 | 变更说明 |
|------|---------|
| `frontend/src/stores/taskStore.ts` | 移除 `isFetching`，保留 `isLoading` 为唯一 loading 标志 |
| `frontend/src/components/TaskCard.tsx` | 用 selector 只订阅 parentProject（不再订阅 tasks 全量数组）；添加 `React.memo` |
| `frontend/src/pages/home/TodayTaskItem.tsx` | 添加 `React.memo` |
| `frontend/src/pages/tasks/DecisionCard.tsx` | 添加 `React.memo` |
| `frontend/src/pages/tasks/ProjectCard.tsx` | 添加 `React.memo` |
| `frontend/src/testUtils/taskStoreHelpers.ts` | 移除 `isFetching` 重置 |

## 验收标准对照

- [x] taskStore 移除 isFetching，保留 isLoading 为唯一 loading 标志
- [x] Tasks.tsx 使用 taskStore.isLoading（已在用，无需改动）
- [x] Explore 页面使用 hook 自有的 isLoading（不依赖 taskStore.isFetching，无需改动）
- [x] Home.tsx 使用 useHomeData（不直接使用 isFetching，无需改动）
- [x] TaskCard 不再订阅整个 tasks 数组，改用 selector 只获取 parentProject
- [x] TaskCard、TodayTaskItem、DecisionCard、ProjectCard 使用 React.memo 包裹
