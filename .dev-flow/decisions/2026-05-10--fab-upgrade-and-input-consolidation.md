# FAB 升级与输入入口整合

## 背景

移动端 FAB（QuickCaptureFAB）当前只能记录灵感，功能单一。同时各 tab 存在多个输入入口，功能有重叠。需要升级 FAB 并整合输入入口，消除冗余。

## 当前输入入口

| 入口 | 位置 | 功能 |
|------|------|------|
| FAB（QuickCaptureFAB） | 全局浮动按钮 | 只能创建灵感 |
| Today 命令栏 | Today 页底部 | AI 判断意图，内联结果（R053 交付） |
| Inbox 快速输入 | Inbox 页底部 | 直接创建灵感条目 |
| Chat 对话输入 | Chat 页底部 | 日知深度对话 |

## 方案对比

### 方案 A: AI 命令模式

- 思路：FAB 点击直接弹出 AI 命令输入框，所有操作通过 AI 意图识别完成
- 优势：交互统一，一个入口覆盖所有
- 劣势：记个灵感也要等 AI 判断，简单操作变重；网络不可用时完全不可用
- 适用条件：AI 响应极快且离线场景少

### 方案 B: 多类型直接创建

- 思路：FAB 展开为多个子按钮（灵感、任务、笔记等），每个走直接创建
- 优势：简单直接，无 AI 依赖
- 劣势：只能做预设类型，无法覆盖"回答问题"等场景
- 适用条件：用户需求明确且可枚举

### 方案 C: 混合模式（快速操作 + AI 入口）

- 思路：FAB 展开后提供：直接记灵感、直接建任务、AI 智能创建（调用 commandBarProvider）
- 优势：常用操作秒级完成，复杂需求走 AI；兼顾速度和灵活性
- 劣势：交互稍复杂，需要合理设计展开后的选项布局
- 适用条件：通用方案

## 决策

- 选择：方案 C（混合模式）
- 理由：方案 A 对简单操作过重，方案 B 对复杂需求能力不足，方案 C 兼顾两者
- 前提条件：commandBarProvider 已就绪（R053 已交付），QuickActions 组件已有参考实现
- 风险：展开交互设计需要用户验证

## 输入入口整合决策

| 入口 | 决策 | 理由 |
|------|------|------|
| FAB | 升级为方案C（灵感+任务+AI命令） | 全局快速入口 |
| Today 命令栏 | 保留，收窄为"今日视角智能操作" | 利用"今日"上下文做差异化 |
| Chat 对话 | 保留 | 深度对话场景完全不同 |
| Inbox 页输入栏 | 移除 | 与 FAB 灵感功能完全重复 |

升级后输入矩阵：
- **全局任意位置**：FAB → 快速灵感 / 快速任务 / AI 智能命令
- **Today 页专属**：命令栏 → 今日视角智能操作（带日期上下文）
- **日知专属**：对话输入 → 深度对话和反思

## 开放问题

- FAB 展开后的 UI 布局方案（子按钮排列、AI 入口的视觉区分）
- Inbox 页输入栏移除后，Inbox 页的页面布局调整
