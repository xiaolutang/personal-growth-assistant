### Acceptance Criteria

1. TokenBlacklist 类实现：add(jti, exp)、is_blacklisted(jti)、cleanup_expired() 方法
2. create_access_token payload 新增 jti 字段（UUID4）
3. TokenData 模型新增 jti 字段，decode_access_token 解析并返回 jti
4. /auth/logout 端点解析 token 获取 jti 和 exp，加入黑名单
5. get_current_user 验证流程新增黑名单检查，命中时返回 401
6. TokenBlacklist 使用 asyncio 定时任务每 10 分钟清理过期记录
7. 黑名单清理任务在 app lifespan startup 中启动、shutdown 中取消
8. 黑名单为空时清理开销为零
9. token 已过期时正常返回 401（不走黑名单逻辑）
10. 多个并发 logout 请求线程安全

### Files to Modify

1. `backend/app/services/auth_service.py` — 新增 TokenBlacklist 类 + 修改 create/decode/get_current_user
2. `backend/app/routers/auth.py` — 修改 /auth/logout 端点
3. `backend/app/routers/deps.py` — get_current_user 加入黑名单检查
4. `backend/app/models/user.py` — TokenData 新增 jti 字段
5. `backend/app/main.py` — lifespan 中启动/取消清理任务

### Contract Ref

- CONTRACT-AUTH01: POST /auth/logout (行为变更)

### Architecture Constraints

- JWT secret 必须通过环境变量配置，不硬编码
- 密码使用 bcrypt 哈希
- 不在日志中记录 token 或密码
- 不做 refresh_token
- 内存黑名单，不引入 Redis
