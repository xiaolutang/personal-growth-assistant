## Code Review

### 修改正确性
- logout() 路径：先调用 authService.logout()（含 _clearAuthData 删除 session_id），再 invalidate chatProvider，最后重置 auth state
- _onApiUnauthorized() 路径：invalidate chatProvider，异步 _clearSessionData()（含 session_id 删除），重置 auth state
- _clearSessionData() 有 try-catch 兜底，SecureStorage 失败不阻塞

### 架构约束遵守
- 清理逻辑统一在 AuthNotifier 层，不依赖页面调用 — 符合 MVVM 分层
- Provider 不含 UI 导航 — 无导航代码
- 认证不变量：session_id 作为聊天隔离的 key，清除后新登录生成新值

### 风险评估
- `late final` → `late` 变更：仅影响 invalidate 后重建场景，正常使用不受影响
- _clearSessionData() 是 fire-and-forget（async void 语义），不等待完成。这是设计决策：401 回调不应阻塞主流程

## Root Cause 修复验证

原始问题：登出/401 流程未清理 chat 相关状态（messages 内存 + session_id 存储）

修复方案：
1. AuthNotifier.logout() — 增加 ref.invalidate(chatProvider)，authService.logout() 内 _clearAuthData 已删除 session_id
2. AuthNotifier._onApiUnauthorized() — 增加 ref.invalidate(chatProvider) + _clearSessionData()（删除含 session_id 的所有认证数据）
3. ChatNotifier.clearMessages() — 新增方法，取消 SSE 连接 + 清空消息状态（作为 invalidate 后的补充，确保资源释放）
