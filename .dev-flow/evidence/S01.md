## Architecture Constraints

- Call direction: routers -> services -> infrastructure, strictly one-way
- SyncService handles three-layer sync (Markdown -> Neo4j -> Qdrant)
- MarkdownStorage is source of truth
- OpenAPI types auto-generated: backend schema -> openapi-typescript -> frontend types

## Acceptance Criteria

1. Task 模型新增 type_history: List[dict] 字段，默认空列表
2. TypeHistoryRecord 包含 from_category、to_category、at（ISO 时间戳）
3. POST /entries/{id}/convert 接收 { target_category, priority?, planned_date?, parent_id? }
4. 只允许合法转换：inbox->task/inbox->decision/inbox->note，其他返回 422
5. 明确禁止的转换：task->reflection（应创建新条目，不走 convert）、task->inbox/decision->task 等反向转换
6. 转换成功后 type_history 追加记录，category 更新，priority/planned_date 按传入值更新
7. MarkdownStorage 读写 type_history（YAML frontmatter），旧文件无此字段时视为空列表
8. EntryResponse 中包含 type_history 字段
9. convert 端点触发 SyncService 三层同步
10. npm run gen:types 成功，前端 api.generated.ts 包含 convert 相关类型和 type_history 字段
