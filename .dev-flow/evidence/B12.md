# B12 执行证据

| Task ID | B12 |
| Phase | P6 |
| Name | 实现 `_default` 历史数据认领与回填服务 |

## 实现结果

- SQLite 新增 `claim_default_entries(target_user_id)`，可将 `_default` 条目批量认领到真实用户。
- Markdown `StorageFactory` 新增 `claim_default_user(target_user_id)`，将 `_default` 用户目录下的历史文件复制到目标用户目录，并保持幂等。
- `SessionMetaStore` 新增 `claim_default_sessions(target_user_id)`，补齐会话元数据认领。
- `SyncService.init_storage()` 启动时先执行 `migrate_default_user()`，把根目录历史 Markdown 收口到 `users/_default`，避免根目录与用户目录长期双轨。
- `EntryService` / `SyncService` / `HybridSearch` 改为按 `user_id` 读取和写入用户目录，不再把根目录当作运行时主数据源。
- 认证服务新增：
  - `claim_default_user_data()` 显式认领服务
  - `auto_claim_default_user_data()` 单用户迁移场景下的安全自动认领
- 认证路由新增：
  - 登录成功后，对“单用户 + 目标账号无数据”的场景自动认领
  - `POST /auth/claim-default-data` 显式认领接口

- 首个真实用户登录时自动认领 `_default` 历史数据
- 多用户场景下不自动认领，避免误归属
- 显式认领接口成功与幂等路径
- SQLite / Markdown / session_meta 三层认领逻辑
- 根目录旧 Markdown 启动迁移到 `users/_default`
- 业务读写路径统一落到用户目录，避免索引和文件源再次分叉
