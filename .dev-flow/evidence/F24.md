## Implementation

### 1. ThemeProvider / useTheme Hook
- **File**: `frontend/src/lib/theme.tsx` (NEW)
- 支持 light / dark / system 三档切换
- localStorage 持久化（key: `rizhi-theme`）
- 监听 `prefers-color-scheme` 变化（system 模式下）
- 通过 `document.documentElement.classList` 切换 `dark` class

### 2. Dark CSS 变量
- **File**: `frontend/src/index.css`
- 新增 `.dark { ... }` 块，覆盖全部 26 个 HSL 变量
- 背景色 → 深灰 (#0a0a0f)，前景色 → 浅色
- 边框/输入/卡片 → 深灰边框，保持层次感

### 3. Header Theme Toggle
- **File**: `frontend/src/components/layout/Header.tsx`
- 替换 Settings 图标为 Sun/Moon 切换按钮
- 使用 `useTheme()` hook 读取当前 resolvedTheme
- 点击切换 light ↔ dark

### 4. MobileNavBar 硬编码修复
- **File**: `frontend/src/components/layout/MobileNavBar.tsx`
- `bg-white border-gray-200` → `bg-background border-t`
- 使用 CSS 变量自动适配深色模式

### 5. App.tsx ThemeProvider 包裹
- **File**: `frontend/src/App.tsx`
- ThemeProvider 包裹在 SidebarProvider 外层
- ProtectedRoute 内有效，Login/Register 不受影响

### 6. 防闪白内联脚本
- **File**: `frontend/index.html`
- `<div id="root">` 后插入 IIFE：读 localStorage → 判断主题 → 注入 `dark` class
- `<html>` 添加 `suppressHydrationWarning`

### 7. Dark 变体批量补齐（17 个文件）
- `constants.ts` — statusConfig, statusIconColor, priorityConfig, categoryConfig
- `intentDetection.ts` — intentConfig 7 种意图颜色

- `Explore.tsx` — 搜索图标、加载文本、错误文本
- `EntryDetail.tsx` — 状态图标
- `Review.tsx` — 统计数字、趋势图标
- `TaskCard.tsx` — 状态图标、项目名、转化菜单
- `OnboardingFlow.tsx` — 背景圆、图标
- `ServiceUnavailable.tsx` — 警告背景、图标
- `OperationStatusBar.tsx` — 成功/错误状态
- `ActionIndicator.tsx` — 成功/错误状态
