## Execution Evidence

### 实现内容

#### 1. 扩展 POST /feedback 请求体
- 添加 `feedback_type` 字段：`"general" | "agent"`（默认 general）
- 添加 `message_id` 字段：Agent 消息 ID（agent 类型时必填，通过 model_validator 校验）
- 添加 `reason` 字段：反馈原因
- 添加 `detail` 字段：详细说明

#### 2. 反馈存储
- `create_feedback` 方法扩展支持 feedback_type / message_id / reason / detail
- SQLite feedback 表添加新列（CREATE TABLE + ALTER TABLE migration）
- FeedbackItem / FeedbackListResponse 等响应模型同步更新

#### 3. Langfuse trace 标记（可选）
- `_langfuse_score_trace()` 函数：通过 message_id 标记对应 trace 评分
- 负面反馈评 0.0，其他评 1.0
- 仅在 LANGFUSE_PUBLIC_KEY + LANGFUSE_SECRET_KEY 均配置时启用
- 失败时静默跳过，不影响核心功能

#### 4. 自动创建 Issue
- `_auto_create_issue_for_negative()` 后台任务
- 👎 负面反馈（reason 属于 `_NEGATIVE_REASONS` 集合）自动创建 Issue
- Issue severity=medium, component=backend
- 标题格式：`[Agent Bad Case] {reason} - {title}`

#### 5. bad case 导出到 Golden Dataset
- `export_to_golden_dataset()` 函数
- 仅导出 agent 类型 + 负面 reason 的反馈
- 输出到 `{DATA_DIR}/eval_transcripts/bad_cases/{message_id}_{timestamp}.json`
- 自动创建目录

### 数据库迁移
- `sqlite_base.py`：CREATE TABLE 中直接包含新列（新数据库）
- `_migrate_feedback_schema()`：ALTER TABLE 为已有数据库添加新列
- 索引：`idx_feedback_type` 加速按类型查询
