# B42: 条目手动关联 API

## Task Snapshot

- **ID**: B42
- **Name**: 条目手动关联 API
- **Phase**: P1
- **Module**: entry-links
- **Priority**: high
- **Verification**: L2
- **Depends on**: (none)

### Acceptance Criteria

1. POST /entries/{id}/links 请求体 { target_id, relation_type } → 201 + { id, source_id, target_id, relation_type, created_at, target_entry }
2. GET /entries/{id}/links[?direction=in|out|both] → { links: [{ id, target_entry, relation_type, created_at }] }
3. DELETE /entries/{id}/links/{link_id} → 204（删除记录及配对记录）
4. 双向关联：A→B 创建时自动创建 B→A，link_id 可使用 GET 返回的任一关联记录 id
5. 不允许自关联 → 400
6. 不允许重复关联 → 409
7. relation_type 非法值 → 422
8. 双向关联写入使用 SQLite 事务保证原子性
9. 删除条目时级联清理 entry_links
10. 所有操作按 user_id 隔离
11. 单元测试覆盖正常/边界/异常场景

### Test Tasks

- 正常：创建 A→B 关联，GET /entries/A/links 和 /entries/B/links 都可见
- 正常：删除 link_id 后双向关联均消失
- 正常：GET /entries/{id}/links?direction=in 只返回指向该条目的关联
- 边界：自关联返回 400
- 边界：重复关联返回 409
- 边界：关联不存在的条目返回 404
- 边界：relation_type 非法值返回 422
- 异常：未认证返回 401
- 级联：删除条目后 entry_links 中相关记录自动清理
- 事务：双向关联写入失败时两行均回滚
