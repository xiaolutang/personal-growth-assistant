## 实现摘要

### 1. KnowledgeService — Neo4j 降级路径统一

- `get_learning_path` 从手动 `if self.is_neo4j_available() + try/except` 改为使用 `_with_neo4j_fallback` 统一降级
- Neo4j 操作包装为 lambda，降级时返回 None，后续 SQLite 补充逻辑不变
- `get_knowledge_graph` 和 `get_related_concepts` 保留纯 Neo4j 端点行为（ValueError → 路由层 503），因为它们没有 SQLite 等价方案
- 其他已有降级路径（get_knowledge_map, get_knowledge_stats, search_concepts, get_concept_timeline, get_mastery_distribution, get_capability_map, get_entry_knowledge_context）已使用 `_with_neo4j_fallback`

### 2. goal_service — JSON 解析去重

- 提取 `_parse_json_fields(self, row)` 方法，就地解析 auto_tags / checklist_items JSON 字段
- `_row_to_response` 和 `_row_to_response_with_current` 都调用 `_parse_json_fields(result)` 替代重复的 if/else 逻辑
- 行为完全不变（纯内部重构）
