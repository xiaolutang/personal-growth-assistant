## Design Decisions

1. **Refresh strategy**: CreateDialog 接受 `skipStoreRefetch` prop。Tasks 页面传入 `skipStoreRefetch` 跳过 store 内部的无参 fetchEntries()，在 onSuccess 回调中用 fetchEntries(TASK_QUERY_PARAMS) 单次精确刷新。其他 CreateDialog 调用方不传此 prop，保持原有行为（store 内部自动刷新）。这消除了双重刷新问题，同时保持了 store API 的向后兼容。

2. **allowedTypes strategy**: 所有子 Tab 的 allowedTypes 统一为 ACTIONABLE_CATEGORIES。子 Tab 仅通过 defaultType 预选对应类型，用户可在任何 Tab 下切换创建类型。

3. **Empty state unification**: 提取 EmptyStateCreateEntry 共享组件，list/grouped/timeline 三种视图的空状态使用相同的创建入口 UI。

4. **Store API extension**: createEntry 新增可选的 `options` 参数而非修改现有签名，保证向后兼容。所有现有调用方（Explore, CompletionPrompt, TypeActionBar, DecisionCard）无需修改。
