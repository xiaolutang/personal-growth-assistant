### 关键设计
1. **同步核心**：sync() 逐条处理 pending 队列项，调用 createEntry() API
2. **防重入**：模块级 `syncing` 布尔锁，同步进行中再次调用被忽略
3. **错误处理**：401 立即标记 failed 并停止（发 auth_failed 事件）；5xx 递增 retry_count，>3 标记 failed
4. **进度通知**：SyncEvent 协议（progress/completed/auth_failed），OfflineIndicator 订阅显示进度
5. **初始化**：initSync() 在 App.tsx 认证校验完成后（isLoading=false）调用
6. **online 事件**：模块顶层注册 window "online" 事件监听，自动触发 sync()
7. **同步后清理**：成功项 queue.remove() + taskStore.removeOfflineEntry()，最后 fetchEntries()
8. **不支持的 mutation**：非 POST /entries 的队列项跳过不删，保留为 pending
