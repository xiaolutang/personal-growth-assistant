# F06 - 页面转场动画

## 变更文件

- `mobile/lib/config/routes.dart`

## 实现内容

### 1. EntryDetail (`/entries/:id`) — Slide 转场
- `builder:` 改为 `pageBuilder:`，使用 `CustomTransitionPage`
- SlideTransition: 从右 (`Offset(1, 0)`) 滑入，Curves.easeInOut

### 2. GoalDetail (`/goals/:id`) — Slide 转场
- `builder:` 改为 `pageBuilder:`，使用 `CustomTransitionPage`
- SlideTransition: 从右 (`Offset(1, 0)`) 滑入，Curves.easeInOut

### 3. Settings (`/settings`) — Fade 转场
- `builder:` 改为 `pageBuilder:`，使用 `CustomTransitionPage`
- FadeTransition: opacity 跟随 animation

### 4. ShellRoute 内主页面 — 保持不变
- `/`, `/chat`, `/explore`, `/tasks`, `/notes`, `/inbox`, `/review`, `/goals`
- 全部保持原有 `builder:` 写法，使用系统默认转场

## 关键设计决策

- 三个详情页路由（EntryDetail、GoalDetail、Settings）均在 ShellRoute 内部
- 使用 `CustomTransitionPage` 而非 `MaterialPageRoute`，与 GoRouter 完全兼容
- 每个页面都设置了 `key: state.pageKey`，确保 GoRouter 正确管理页面生命周期
- 由于在 ShellRoute 内，自定义转场不会与 Shell 动画冲突
- iOS swipe-back 手势：CustomTransitionPage 在 ShellRoute 内由 Shell 的 navigator 管理，手势行为取决于 Shell navigator 的配置，slide 过渡与之兼容
