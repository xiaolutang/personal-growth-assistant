# B46: Goal 条目关联 + 进度计算 + progress-summary

## Task Snapshot

- **ID**: B46
- **Module**: goals
- **Depends On**: B45 (completed)
- **Priority**: high
- **Verification**: L2

### Acceptance Criteria
1. goal_entries 表：id, goal_id, entry_id, user_id, created_at
2. POST /goals/{id}/entries 关联条目到 count 类型目标
3. DELETE /goals/{id}/entries/{entry_id} 取消关联
4. GET /goals/{id}/entries 返回关联条目列表
5. count 类型进度 = 已关联条目数 / target_value（关联即计数）
6. checklist 类型进度 = checked=true 的 item 数 / total items
7. tag_auto 类型进度 = 匹配 auto_tags 且在时间范围内的条目数 / target_value（使用 entry_tags 归一化表）
8. 自动完成：进度达到 100% 时 status 自动更新为 completed
9. 状态不自动回退：进度下降时 status 保持 completed，用户可手动改回 active
10. GET /goals/progress-summary?period=weekly|monthly 返回活跃目标进度概览含 progress_delta
11. 不允许关联到非 count 类型目标
12. 单元测试覆盖三种类型的进度计算 + 401/403

### Test Tasks
- 正常：关联条目到 count 目标，进度更新（关联即计数）
- 正常：取消关联后进度回退
- 正常：checklist 勾选后进度更新
- 正常：tag_auto 根据匹配 tag 的条目数计算进度
- 正常：进度 100% 时自动标记 completed
- 正常：GET /goals/progress-summary 返回正确概览
- 边界：关联到 tag_auto 类型返回 400
- 边界：关联已删除条目返回 404
- 边界：重复关联返回 409
- 异常：未认证返回 401
- 异常：操作他人目标条目关联返回 403
- 异常：取消关联后 count 进度下降但 completed 状态保持不变
