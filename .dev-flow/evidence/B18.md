### 实现要点
- GET /entries/export 支持双格式：markdown (zip) / json
- 路由声明在 /{entry_id} 之前，避免被动态路由吞掉
- 参数：format (默认 markdown)、type (可选，白名单校验)、start_date/end_date (可选，ISO 格式校验)
- markdown 格式：真正流式 — tempfile.mkstemp → 逐条 zipfile.writestr → 8KB 分块 yield → finally os.unlink
- json 格式：JSONResponse，从 SQLite 查询 + EntryMapper 转换
- 所有查询带 user_id 隔离（通过 Depends(get_current_user)）
- 空数据：markdown 返回空 zip，json 返回空数组
- Content-Disposition: entries_export.zip
- 导出上限 100000 条，达到时记录 warning
