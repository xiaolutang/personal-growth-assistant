### 真实 503 降级运行态场景验证 (2026-04-12)

场景：停止后端制造服务不可用，验证前端降级链路

**步骤 1：停止后端服务**
```bash
$ kill $(lsof -ti :18929) 2>/dev/null
# exit: 0
```

**步骤 2：确认 Vite dev server 仍在运行**
```bash
$ lsof -i :5173 -P -n 2>/dev/null | grep LISTEN
node    18965 tangxiaolu   13u  IPv6 ... TCP [::1]:5173 (LISTEN)
```

**步骤 3：验证前端页面可达（SPA 总是返回 index.html）**
```bash
$ curl -s -o /dev/null -w "%{http_code}" http://localhost:5173/growth/tasks
200
```

**步骤 4：验证 API 不可用（后端已停止）**
```bash
$ curl -s -w "\nHTTP_CODE:%{http_code}\n" http://localhost:5173/api/entries 2>&1
HTTP_CODE:500
```

> 注意：后端完全停止时，Vite proxy 连接失败返回 HTTP 500 而非 503。这是因为 Vite http-proxy 在 ECONNREFUSED 时统一返回 500。前端 503 降级检测（`status === 503`）不会在此场景触发。
>
> 503 的正确触发路径是：后端进程存活但存储层未初始化（storage=None）→ 后端主动返回 HTTP 503 + "存储服务未初始化" → 前端检测到 503 → 渲染降级组件。
>
> 这是运行态验证的重要发现：**503 降级仅在后端自身返回 503 时生效，不覆盖后端完全停止（ECONNREFUSED）场景**。

**步骤 5：前端 503 检测链路（代码追踪）**

| 环节 | 文件 | 行为 |
|------|------|------|
| API 调用 | `frontend/src/lib/errors.ts:101-126` | `handleApiResponse` 检测非 2xx 响应，抛出 `ApiError(response.status, message)` |
| 503 识别 | `frontend/src/lib/errors.ts:58-60` | `ApiError.isServiceUnavailable` getter：`return this.status === 503` |
| Store 状态 | `frontend/src/stores/taskStore.ts:71-74` | `fetchEntries` catch 中检测 `error.isServiceUnavailable`，设置 `serviceUnavailable=true` |
| UI 降级 | `frontend/src/pages/Tasks.tsx` | 当 `serviceUnavailable=true` 时渲染 `<ServiceUnavailable onRetry={fetchEntries}>` |
| 降级组件 | `frontend/src/components/ServiceUnavailable.tsx` | 显示"服务暂时不可用"文案 + 重试按钮 |

**步骤 6：重启后端恢复服务**
```bash
$ cd backend && uv run uvicorn app.main:app --port 18929 &>/tmp/growth-backend.log &
sleep 3
```

**步骤 7：验证恢复后 API 正常**
```bash
$ curl -s -o /dev/null -w "%{http_code}" http://localhost:5173/api/entries
200
```

**结论**：
- curl http://localhost:5173/growth/tasks -> 200（SPA 页面始终可达）
- curl http://localhost:5173/api/entries（后端停止时）-> 500（Vite proxy ECONNREFUSED，非 503）
- 503 降级 UI 是 JS 条件渲染，curl 无法直接看到 DOM 变化
- 前端 503 检测链路已通过 6 个单元测试验证（3 store + 3 component）
- 运行态发现：Vite proxy 连接失败返回 500，当前 503 降级不覆盖此场景；后端完全停止时前端会显示普通错误而非降级 UI
