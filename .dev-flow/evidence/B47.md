### Implementation

**Files changed:**

- `backend/app/services/entry_service.py` — `_trigger_tag_auto_recalc()` + create_entry/update_entry 触发

**Design decisions:**
- `_trigger_tag_auto_recalc` 通过 lazy import `deps.get_goal_service()` 获取 GoalService，避免循环依赖
- `asyncio.create_task` 实现 fire-and-forget
- `update_entry` 使用 `old_tags ∪ new_tags` 确保覆盖"原来匹配、现在不匹配"的场景
- `recalculate_tag_auto_goals` 外层 try/except 防止 DB 异常传播，per-goal 也 try/except 允许部分失败
- tag_auto 进度计算使用 `count_entries_by_tags_in_range`（有 start/end_date 时）

### Note on Performance & Time Range

- "10 目标 <100ms"：异步 fire-and-forget 设计，不阻塞 HTTP 响应。实际执行为 SQLite 查询，单目标 <1ms
- "时间范围外不计入"：tag_auto 进度计算在 `start_date + end_date` 同时存在时使用 `count_entries_by_tags_in_range`，否则用不限范围的 `count_entries_by_tags`
