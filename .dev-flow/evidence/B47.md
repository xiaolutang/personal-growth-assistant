# B47: Goal 自动追踪触发

## Task Snapshot

- **ID**: B47
- **Module**: goals
- **Depends On**: B45 (completed)
- **Priority**: medium
- **Verification**: L2

### Acceptance Criteria
1. entry_service.create_entry 完成后，触发 tag_auto 目标进度重算
2. entry_service.update_entry 修改 tags 后，触发 tag_auto 目标进度重算
3. 重算逻辑：查找所有 metric_type='tag_auto' 且 status='active' 或 status='completed' 且 auto_tags 与条目更新前后 tags 有交集的目标
4. 重算为异步 fire-and-forget，不阻塞条目创建/更新响应
5. 重算失败不影响条目操作（catch + log warning）
6. 单元测试覆盖触发场景

### Test Tasks
- 正常：创建带 tag 的条目后，匹配的 tag_auto 目标进度更新
- 正常：更新条目 tags 后，匹配的 tag_auto 目标进度更新（含原来匹配现在不匹配的回退）
- 正常：条目 tags 不匹配任何目标时不触发重算
- 边界：目标时间范围外的条目不计入
- 异常：重算失败时条目操作仍成功
- 性能：10 个活跃目标时重算耗时 < 100ms

### Implementation

**Files changed:**
- `backend/app/services/goal_service.py` — 新增 `recalculate_tag_auto_goals()` 方法（53 行）
- `backend/app/services/entry_service.py` — 新增 `_trigger_tag_auto_recalc()` + 在 `create_entry`/`update_entry` 中触发

**Design decisions:**
- `_trigger_tag_auto_recalc` 通过 lazy import `deps.get_goal_service()` 获取 GoalService，避免循环依赖
- `asyncio.create_task` 实现 fire-and-forget
- `update_entry` 使用 `old_tags ∪ new_tags` 确保覆盖"原来匹配、现在不匹配"的场景
- `recalculate_tag_auto_goals` 外层 try/except 防止 DB 异常传播
