## Implementation Summary

为 Explore 页面的 inbox/reflection/question tab 添加了 '+New' 创建入口：

1. **新增导入**: `Plus` 图标、`useState`、`CreateDialog` 组件、`Category` 类型
2. **状态管理**: `createDialogOpen` / `createDialogType` 控制 CreateDialog 显隐和默认类型
3. **显示条件**: `showCreateButton = !isSearchMode && !searchQuery.trim() && TABS_WITH_CREATE.has(activeTab)`
   - 仅 inbox/reflection/question 三个 tab 显示
   - 搜索态（含搜索失败）和全部 tab 不显示
   - 通过 `searchQuery.trim()` 确保搜索失败（searchResults=null 但有搜索词）时也不显示
4. **CreateDialog 配置**: `defaultType={createDialogType}`, `allowedTypes={[createDialogType]}`, `skipStoreRefetch={true}`
   - `allowedTypes` 约束为单一类型，用户无法切换到其他分类
   - `skipStoreRefetch=true` 避免 taskStore 全局刷新，由 `onSuccess` 回调调用 `loadEntries()` 刷新本地列表
5. **按钮位置**: CardHeader 区域，独立于编辑按钮条件，空列表时也可见
