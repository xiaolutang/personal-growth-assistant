### Acceptance Criteria

1. 同一用户同一天第二次请求命中缓存，不触发数据库查询和LLM调用
2. 新的一天首次请求不命中缓存，正常计算
3. 缓存命中时响应包含 cached_at 时间戳字段（ISO格式）
4. 缓存未命中时 cached_at 为 null
5. 缓存未命中时行为与现有完全一致（全量回归通过）
6. 并发安全：同一用户同日并发首请求只计算一次（asyncio.Lock）
7. 旧缓存 key 在跨日时自动清理（LRU 上限 1000 条）

### Test Tasks

1. 缓存命中：mock 底层方法，第二次调用不触发
2. 缓存失效：模拟跨日，缓存失效重新计算
3. cached_at 字段：命中时有值，未命中时为 null
4. 并发去重：同一用户并发首次请求，底层只调用一次
5. 缓存清理：模拟缓存超过上限，旧条目被淘汰
6. 回归：现有晨报测试全部通过
