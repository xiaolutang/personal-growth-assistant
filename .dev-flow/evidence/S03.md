## Acceptance Criteria

1. runtimeCaching URL pattern 使用完整 URL 正则匹配 API 请求（含 query string）
2. 条目列表 GET /entries 使用 NetworkFirst（5min TTL）
3. 条目详情 GET /entries/{id} 使用 StaleWhileRevalidate
4. 搜索 POST /search 和 SSE 流式接口使用 NetworkOnly
5. 构建后 Offline 模式刷新不白屏

## Architecture Constraints

- SW navigateFallback 保持 `index.html`（支持 SPA deep-link）
- 不修改 navigateFallbackDenylist
- base path 可能为 `/growth/` 或其他，URL pattern 需兼容

### 改动文件

### 改动内容
将原来的单一 `runtimeCaching` 规则（错误路径正则 `/^\/growth\/api\/.*/i`）替换为 4 条分层规则：

| # | 匹配目标 | Handler | URL Pattern |
|---|---------|---------|-------------|
| 1 | SSE/搜索/认证/反馈/意图/调试 | NetworkOnly | `/(ai/chat\|chat\|parse\|search\|auth\|feedback\|intent\|playground\|entries/search\|entries/export)(/\|$?\?)` |
| 2 | 条目详情+子资源 /entries/{id}(/sub)? | StaleWhileRevalidate | `/entries/[^/]+(/[^/]+)?(\?.*)?$` |
| 3 | 条目列表 /entries | NetworkFirst 5min | `/entries(\?.*)?$` |
| 4 | 其他 API 兜底 | NetworkFirst 5min | `/api/` |

### 关键修复
1. **URL pattern 从路径匹配改为完整 URL 匹配**：Workbox 匹配完整 URL，正则改为匹配路径部分在完整 URL 中的位置
2. **动态 base path**：`apiRe` 从 `process.env.FRONTEND_BASE_PATH` 动态构建，不硬编码 `/growth/`
3. **NetworkOnly 覆盖子路径**：使用 `(/|$|\\?)` 后缀匹配，确保 `/auth/login`、`/feedback/123` 等子路径也被拦截
4. **排除 export/search**：规则 1 先拦截 `/entries/export` 和 `/entries/search/*`，规则 2 不会误匹配
