Classification Dimensions

Each test case is classified into a category (e.g., tool_selection, no_tool, boundary). Categories map to common user intent patterns the Agent should handle.

Pass/Fail Criteria

  • Positive case: PASS if actual_tools matches expected_tools; FAIL otherwise.
  • Negative case: PASS if Agent does NOT call any tool in should_not_call; VIOLATED otherwise.
